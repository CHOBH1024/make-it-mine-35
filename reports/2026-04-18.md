## 1. 프로젝트 개요

이 프로젝트는 가정연합(통일교) 목회자와 사역자를 대상으로, 자신의 사역 유형을 심리학적·섭리적 관점에서 진단하고 이해하도록 돕는 웹 애플리케이션이다. 단순한 성격 검사가 아니라, 실제 사역 현장에서의 행동 패턴, 감정 지능, 가치 우선순위, 강점을 복합적으로 측정하여 9가지 목회자 아키타입 중 자신에게 해당하는 유형을 도출하는 구조다.

기술 스택은 React 19 + TypeScript + TanStack Router + TailwindCSS + Vite 조합으로 구성되어 있으며, Lovable 플랫폼을 통해 자동 배포된다. GitHub 레포지토리의 main 브랜치에 코드가 머지되면 배포가 자동으로 이루어진다.

전체 작업은 5인 가상 실무팀 체계로 운영된다. 우성(인프라), 지성(리서치), 짱구(기획·테스트), 김치워리어(PM·코드감사), 서진(총괄·배포)이 각자의 역할에 따라 분업하여 작업을 진행한다.

## 2. 진단 도구 현황 (5개 완료)

### 2-1. 에니어그램 (Enneagram)

에니어그램은 인간의 성격을 9가지 유형으로 분류하는 심리 모델이다. 이 앱에서는 사용자가 외부 에니어그램 검사(예: 공식 에니어그램 협회 검사)를 먼저 완료한 뒤, 그 결과를 여기에 입력하는 방식으로 사용된다.

구현 내용:
- 1번부터 9번까지 유형 선택 버튼 UI
- 각 유형 버튼에 핵심 키워드 태그 추가. 예를 들어 1유형은 "원칙·완벽", 2유형은 "돌봄·관계", 3유형은 "성취·이미지", 4유형은 "독창·감수성", 5유형은 "탐구·고독", 6유형은 "충성·안전", 7유형은 "열정·자유", 8유형은 "주도·강인", 9유형은 "평화·조화"로 표시된다.
- 결과 해석 가이드 접이식 섹션: "검사 점수가 가장 높은 유형 번호를 선택하세요", "날개(Wing)가 있다면 주 유형 기준" 등 안내 문구 포함
- 선택한 유형 번호가 상태(inputs.enneagram)에 저장되어 아키타입 매칭에 활용됨

### 2-2. Big 5 성격 특성 (OCEAN)

Big 5는 심리학에서 가장 널리 사용되는 성격 모델로, 개방성(Openness), 성실성(Conscientiousness), 외향성(Extraversion), 친화성(Agreeableness), 신경성(Neuroticism) 다섯 가지 차원을 측정한다. 앱에서는 bigfive-test.com 등의 외부 검사 결과를 입력하는 방식이다.

구현 내용:
- 5개 특성 각각에 대해 High / Mid / Low 세 가지 레벨 선택 버튼
- 각 특성 섹션 상단에 점수→레벨 변환 기준 인라인 표시: "60% 이상 → High / 40~60% → Mid / 40% 미만 → Low"
- 팁 박스: "bigfive-test.com 결과 기준 백분위 점수를 위 기준으로 변환하세요" 안내
- 선택 결과가 inputs.big5 객체에 저장됨

### 2-3. 커리어 앵커 (Career Anchor)

에드가 샤인(Edgar Schein)이 개발한 커리어 앵커는 개인이 직업적 정체성에서 절대 포기할 수 없는 가치를 8가지로 분류한 모델이다. 목회 사역 맥락에서 각 앵커는 사역자가 어떤 방식으로 부르심을 이해하는지를 나타낸다.

구현 내용:
- 8개 앵커 버튼: 전문가적 역량, 총괄 관리 역량, 안전·안정, 기업가적 창의성, 자율·독립, 봉사·헌신, 순수한 도전, 생활 방식
- 각 앵커 버튼에 핵심 질문 한 줄 힌트 표시. 예: "전문가적 역량" 버튼 아래에 "나는 특정 분야의 깊은 전문성을 인정받을 때 가장 보람을 느낀다"
- 팁 박스: "점수 1위 앵커를 선택하세요. 동점이면 '이것을 잃으면 안 된다'는 항목을 선택하세요"
- 선택 결과가 inputs.anchor에 저장됨

### 2-4. VIA 강점 (Values in Action)

VIA 강점 검사는 크리스토퍼 피터슨과 마틴 셀리그먼이 개발한 긍정심리학 기반 모델로, 24가지 성격 강점을 측정한다. 사역자가 가장 자연스럽게 발휘하는 강점 5개를 선택한다.

구현 내용:
- 24개 강점 버튼 목록 전체 표시
- 상단에 실시간 검색 필터 입력창 추가: 입력한 키워드로 강점 목록을 즉시 필터링
- 선택 카운터: 현재 몇 개 선택했는지 표시 (최대 5개)
- 5개 초과 선택 방지 로직
- 팁 박스: "viacharacter.org 결과 상위 5개 강점을 선택하세요"
- 선택 결과가 inputs.via 배열에 저장됨

### 2-5. EQ 감성지능 (Emotional Intelligence) — 앱 자체 진단

EQ는 외부 검사를 참조하지 않고, 앱 안에서 직접 25문항을 답변하는 방식으로 진단한다. 다니엘 골먼(Daniel Goleman)의 EQ 모델을 바탕으로 5가지 차원을 측정한다.

5개 차원:
1. 자기인식(Self-Awareness): 자신의 감정 상태를 인지하고 명명하는 능력
2. 자기조절(Self-Regulation): 감정을 상황에 맞게 조절하고 충동을 통제하는 능력
3. 동기부여(Motivation): 외부 보상 없이도 내적 동기로 행동하는 능력
4. 공감(Empathy): 타인의 감정과 필요를 직감적으로 파악하는 능력
5. 사회적 기술(Social Skills): 관계를 형성하고 갈등을 해결하는 능력

각 차원별 5문항, 총 25문항. 모든 문항은 가정연합 사역 현장 시나리오로 작성되었다.

예시 문항 (자기인식 차원):
- "힘든 심방을 마친 직후 내 감정 상태가 어떤지 즉시 알아차린다"
- "사역 중 화가 날 때 그 원인이 무엇인지 정확히 파악한다"
- "내 감정이 식구들을 대하는 방식에 어떤 영향을 미치는지 인식한다"

구현 내용:
- 3점 척도 버튼: "전혀 아니다(1)" / "보통이다(2)" / "매우 그렇다(3)"
- 답변 완료된 문항은 배경색 강조
- 상단 전체 진행도 바: 25문항 중 몇 개 답변했는지 시각적으로 표시
- 차원별 자동 레벨 산출:
  - 5문항 합계 13~15점 → High
  - 8~12점 → Mid
  - 7점 이하 → Low
  - 미답변 항목 있으면 미완료
- useEffect로 답변 변화 감지 시 inputs.eq 자동 갱신

## 3. 아키타입 데이터 확장 (9개 유형 전면 개편)

진단 결과를 토대로 사용자를 9가지 목회자 아키타입 중 하나로 매칭한다. 각 유형의 데이터는 이번 작업에서 단문 키워드에서 상세 서술로 전면 확장되었다.

확장된 필드 구조:
- dna.how: 이 유형이 사역하는 방식 (2~3문장)
- dna.what: 이 유형이 담당하는 구체적 역할 (2~3문장)
- dna.why: 섭리적 소명과 동기 (2~3문장, 참부모님 말씀 기반)
- synergyDesc: 조직 내 시너지 기여 방식 (3문장)
- activities: 주요 활동 4항목 (사역 맥락 포함 상세 설명)

### 유형 1 — 천일국 설계자
비전 수립과 체계적인 계획으로 팀을 이끄는 유형. 원칙을 중시하고 완벽을 추구하며, 섭리적 구조와 시스템을 설계하는 데 탁월하다. 사역의 방향성을 잡고 장기 로드맵을 수립하는 역할을 담당한다.

### 유형 2 — 심정의 부모
식구들의 필요를 먼저 파악하고 진심으로 돌보는 유형. 관계와 연결을 통해 공동체를 따뜻하게 묶어주는 역할을 한다. 참부모님의 심정을 가장 직접적으로 체현하는 사역 스타일이다.

### 유형 3 — 성취하는 전략가
명확한 목표를 설정하고 가장 효율적인 전략으로 팀을 이끄는 유형. 감정이나 분위기에 흔들리지 않고 데이터와 성과 지표를 기반으로 의사결정하며, 결과로 사역의 가치를 증명한다. 천일국 승리를 위한 실질적 돌파구를 여는 역할이다.

### 유형 4 — 천일국 살림꾼
공동체의 일상을 세심하게 챙기고 지속가능한 사역 환경을 조성하는 유형. 화려한 성과보다 묵묵한 섬김을 통해 조직의 뿌리를 깊게 내리는 역할을 담당한다. 겉으로 드러나지 않지만 없으면 무너지는 기둥 같은 존재다.

### 유형 5 — 진리 탐구자
깊은 연구와 통찰로 말씀의 본질을 탐구하는 유형. 원리·신학·철학적 기초를 다지고, 사역의 지적 토대를 공급하는 역할을 한다. 빠른 행동보다 정확한 이해를 우선시하며, 혼자 깊이 파고드는 사색의 시간을 통해 중요한 통찰을 가져온다.

### 유형 6 — 충직한 수호자
신뢰와 충성을 바탕으로 공동체를 지키는 유형. 리더에 대한 헌신과 원칙에 대한 철저한 준수가 특징이다. 위기 상황에서 흔들리지 않고 조직의 안전망 역할을 수행하며, 신뢰 관계 형성에 탁월하다.

### 유형 7 — 기쁨의 전파자
열정적인 에너지와 긍정적인 비전으로 사람들에게 희망을 불어넣는 유형. 새로운 아이디어와 가능성을 끊임없이 탐색하며, 축제와 기쁨의 분위기를 창출한다. 어떤 상황에서도 섭리적 낙관주의를 잃지 않는다.

### 유형 8 — 용맹한 개척자
두려움 없이 새로운 영역을 개척하고 불의에 맞서는 유형. 강한 의지와 실행력으로 아무도 가지 않는 길을 먼저 열어낸다. 타협보다 원칙, 안주보다 돌파를 선택하는 사역 스타일이다.

### 유형 9 — 평화의 중재자
갈등을 조율하고 다양한 입장을 통합하는 유형. 모든 사람의 목소리에 귀 기울이며, 분열된 공동체를 하나로 연결하는 역할을 한다. 조용하지만 깊은 영향력으로 팀의 화합을 이끈다.

### 4-3. 정밀 보고서 발송 (send-full-report.yml)

### 공통 이메일 발송 방식

초기에는 Resend API(외부 이메일 서비스)를 사용하려 했으나, GitHub Actions 실행 환경에서의 네트워크 제한으로 전환을 검토했다. 최종적으로 Python 표준 라이브러리인 smtplib를 사용한 Gmail SMTP 방식으로 확정했다.

```python
with smtplib.SMTP_SSL('smtp.gmail.com', 465) as server:
    server.login('nokira1024@gmail.com', pw)
    server.sendmail('nokira1024@gmail.com', ['nokira1024@gmail.com'], msg.as_string())
```

Gmail 앱 비밀번호(App Password)는 GitHub Repository Secret(GMAIL_APP_PASSWORD)으로 관리된다. 복사-붙여넣기 시 non-breaking space(\xa0) 문자 혼입 문제를 방지하기 위해, 코드 내에서 패스워드 클렌징 처리를 적용했다:

```python
pw = os.environ['GMAIL_APP_PASSWORD'].encode('ascii', 'ignore').decode('ascii').replace(' ', '').strip()
```
